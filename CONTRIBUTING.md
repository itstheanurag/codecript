# Contributing to Codecript

Thank you for your interest in contributing to Codecript! This project aims to be a comprehensive resource for software engineering job preparation, and your contributions help make it better for everyone.

## How to Contribute

We welcome contributions in several forms:

### 1. Adding New Content

If you have a document, guide, or tutorial that would be helpful for software engineering interviews:

- Ensure the content is in **Markdown** format.
- Place it in the appropriate category under the `content/` directory.
- Use clear headings and code blocks where applicable.

### 2. Improving Existing Content

If you find a typo, outdated information, or a better way to explain a concept:

- Feel free to submit a PR with the improvements.
- Keep the language clear and concise.

### 3. Code Contributions

While the primary focus is content, we also welcome improvements to the project's structure or features:

- Follow the existing project structure (React + TypeScript + Vite).
- Ensure your code is well-commented and follows clean code principles.

## Pull Request Process

1. **Fork the repository** and create your branch from `main`.
2. **Make your changes**.
3. **Ensure your changes follow the documentation style**.
4. **Submit a Pull Request** with a clear description of what you've added or changed.
5. **Wait for review**. We'll get back to you as soon as possible!

## Content Guidelines

- **Accuracy**: Ensure the information provided is technically correct.
- **Clarity**: Write in a way that is easy to understand for developers of all levels.
- **Relevance**: Content should be directly related to software engineering job preparation (algorithms, system design, behavioral, etc.).

By contributing to Codecript, you agree that your contributions will be licensed under the project's LICENSE.
